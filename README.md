# Android PWA PDF Viewer 仕様

## 概要

GitHub Pages で公開する、Android 向けの PWA ベース PDF Viewer を実装する。
スマートフォンでの閲覧を主用途としつつ、タブで複数の PDF を同時に開いて切り替えられることを前提にする。
公開用の静的ファイルは `docs/` 配下に配置し、GitHub Pages の配信対象にする。

## 公開 URL

- GitHub Pages 想定 URL: `https://airpocket-soundman.github.io/pdf_viewer/`
- 上記は GitHub アカウント `airpocket-soundman` とリポジトリ名 `pdf_viewer` を前提にした Project Pages の URL

## ゴール

- Android Chrome でホーム画面追加できる PWA として動作する
- ローカルの PDF ファイルを複数読み込み、タブで切り替えられる
- 基本表示はフルスクリーン前提の閲覧 UI とする
- GitHub Pages にそのまま配置できる静的ファイル構成にする

## 主要機能

### 1. PDF 読み込み

- `input[type="file"]` から 1 つ以上の PDF を選択できる
- 複数選択したファイルは同時にタブとして追加される
- 同名ファイルでも別タブとして開ける

### 2. タブ UI

- 画面上部に横スクロール可能なタブバーを配置する
- アクティブタブが明確に分かる見た目にする
- 各タブは閉じられる
- 1 つの PDF を表示中でも別の PDF をバックグラウンドで保持する

### 3. PDF 表示

- 縦スクロール型のビューアで全ページを連続表示する
- 表示領域はフルスクリーンを優先し、ブラウザ UI を除く領域を最大限使う`r`n- ページ数が少ない場合は表示領域を内容に応じた高さに保ち、余計な縦スクロールを出さない`r`n- 新規に開いた PDF は初期状態で Fit 表示にする
- 現在ページ番号と総ページ数を表示する
- 前後ページ移動を提供する
- ズームイン、ズームアウト、幅合わせを提供する
- 必要に応じて 20% まで縮小でき、Fit で画面幅に収まる倍率まで下げられる

### 4. 状態保持

- 開いているタブ一覧を IndexedDB に保存する
- ファイル本体とタブ順、アクティブタブ、ズーム状態を保存する
- 再読み込み後に前回セッションのタブを復元する

### 5. PWA

- `manifest.webmanifest` を用意する
- `service worker` でアプリシェルをキャッシュする
- GitHub Pages 配下で動作する相対パス構成にする
- オフライン時でもアプリ UI は起動可能にする
  - ただし初回ロード前に取得していない外部依存は利用できない

## UI 方針

- Android の片手操作を意識したコンパクトなヘッダー
- 背景はダークトーン、PDF 面は高コントラストで視認性を確保する
- 上部にアプリバーとタブバー、下にスクロールビューアを置く
- 空状態では操作方法が分かる導線を表示する

## 技術方針

- ビルド不要の静的 HTML/CSS/JavaScript で構成する
- PDF 描画は `pdf.js` を利用する
- 永続化は IndexedDB を利用する
- GitHub Pages 互換のため、クライアントサイドのみで完結させる

## 非対象

- 注釈編集
- PDF への書き込み
- クラウド同期
- サーバーサイド処理
- 高度な全文検索
